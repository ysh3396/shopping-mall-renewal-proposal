IMWEB OVERVIEW

아임웹에서 독립몰로, 왜?

현재: 아임웹 호스팅

아임웹은 임대 아파트와 같습니다.
입주(가입)하면 바로 쇼핑몰을 운영할 수 있지만,
벽을 허물거나 구조를 바꿀 수는 없습니다.

✗ 기능 확장 거의 불가 — 커스텀 개발 제약이 큼
✗ SEO 제약 — 메타 태그/URL 구조 제어 한계
✗ 데이터(회원/주문)를 직접 소유하지 못함
✗ 성능 최적화 불가 — 이미지/코드/캐싱 직접 제어 X
전환 후: 클라우드 독립몰 (자가 소유)

독립몰은 내 건물을 짓는 것입니다.
원하는 구조로 자유롭게 설계하고,
모든 데이터를 직접 소유합니다.

✓ 원하는 기능 무제한 추가/커스터마이징 가능
✓ SEO 완전 제어 — 검색 유입 극대화
✓ 회원/주문/매출 데이터 100% 소유
✓ AWS 서버리스 — 호스팅 서버 없이 자동 스케일링

이관 과정에서 기존 회원 정보, 상품, 주문 내역은 모두 새 시스템으로 안전하게 옮겨집니다. 고객은 변화를 거의 느끼지 못합니다.

MIGRATION RISKS

이관 시 저희가 챙겨드릴 부분

아임웹에서 독립몰로 전환할 때 발생할 수 있는 이슈를 미리 파악하고, 안전하게 처리합니다.

📦

데이터 이관

  • 회원/상품/주문 데이터를 아임웹 Open API v3 + Excel Export로 추출하여 새 시스템으로 옮깁니다
  • 기존 비밀번호는 보안상 가져올 수 없어 임시 비밀번호를 발급하고, 최초 로그인 시 변경을 안내합니다
  • 사용자 정의 회원 필드는 일괄 내보내기 미지원 — API로 개별 추출합니다
🖼️

상품 이미지

  • 아임웹 이미지는 cdn.imweb.me에 저장되며 데이터 백업에 이미지 미포함
  • 공식 일괄 다운로드 기능 없음 — API로 URL 추출 후 자동화 스크립트로 수집합니다
  • 새 S3 + CloudFront로 이미지를 옮기고, 모든 링크를 자동 교체합니다
💳

결제(PG) 계약

  • 아임웹에 한 사이트 하나의 PG만 연동 가능 — 현재 PG사 확인 필수
  • KG이니시스/토스페이먼츠는 기존 MID 추가 발급 가능, 네이버페이/카카오 체크아웃은 신규 신청 필요
  • PG 심사에 1~2주 소요 — 미팅 직후 접수 필요
🌐

도메인 전환

  • 아임웹 도메인 이전은 고객지원 채팅으로만 요청 가능 (Auth Code 발급 1~2 영업일)
  • 국제 도메인(.com)은 이전 후 60일간 재이전 불가 (ICANN 정책)
  • 전환 중에도 아임웹을 병행 운영하여 안전하게 이관합니다

핵심 질문

Q1. 도메인 등록 기관

도메인을 아임웹에서 구매했는지, 가비아/후이즈 등 외부에서 구매했는지? 아임웹 구매 시 고객지원 통해 Auth Code 발급 필요 (1~2 영업일)

도메인 이메일(info@도메인.com 등) 사용 중인지? DNS 변경 시 이메일 수신 즉시 중단 위험 — MX 레코드 보존 필수

Q3. SSL 인증서 현황

아임웹 SSL은 이전 불가 — 새 플랫폼(AWS)에서 ACM으로 무료 재발급. 별도 인증서 구매한 것이 있는지?

Q4. 외부 채널 URL

네이버 쇼핑/다나와 등에 등록된 상품 URL이 있는지? URL 구조 변경 시 301 리다이렉트 매핑 필요

전달받을 자료

도메인 등록기관 로그인 정보 현재 DNS 레코드 목록 SSL 인증서 현황 도메인 만료일 정보

미확인 시 리스크

치명적: DNS 변경 시 MX 레코드 초기화 → 비즈니스 이메일 수신 즉시 중단

치명적: 아임웹 구매 도메인은 셀프서비스 이전 불가 — 반드시 고객지원 요청 필요. 기관이전 중 DNS 변경 불가

현재 가입 회원 수? 활성 회원 비율? → 이관 볼륨 및 DB 용량 산정에 필요

P0비밀번호 재설정 안내 방법

비밀번호는 암호화되어 이전 불가. 이관 후 회원에게 비밀번호 재설정 안내 이메일/SMS 발송 필요. 발송 방법 협의 (카카오 알림톡 vs 이메일 vs SMS)

P1쇼핑 등급/포인트 사용 여부

회원 등급 체계가 있는지? 적립금/포인트 잔액 이관 필요? API로 포인트 거래 내역까지 추출 가능

P1로그인 방식

이메일 로그인? 소셜 로그인(카카오/네이버/구글)? 아이디 로그인? → 새 시스템 인증 구조 설계에 영향

이관 가능/불가 항목

✓ 이메일, 이름, 전화번호, 주소
✓ 쇼핑 등급, 회원 그룹
✓ 적립금/포인트 잔액
✓ 동의 정보 (개인정보/마케팅)
✓ 위시리스트, 장바구니 (API)
✗ 비밀번호 (암호화 — 이전 불가)
✗ 사용자 정의 필드 (일괄 내보내기 미지원)
✗ 회원 첨부 이미지/파일 (백업 미포함)

비밀번호 재설정 안내 미발송 시 회원 로그인 불가 → 이탈 위험. 이관 후 즉시 안내 발송 필수

사용자 정의 필드가 있을 경우 일괄 내보내기 불가 — API로 개별 회원 조회하여 추출해야 함 (대량이면 시간 소요)
