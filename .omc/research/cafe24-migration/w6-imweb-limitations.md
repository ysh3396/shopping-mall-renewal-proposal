## 1. 템플릿 기반 구조의 근본적 제약

아임웹은 노코드 빌더 특성상 **템플릿 기반 구조**로 설계되어 있어, 전자담배 쇼핑몰처럼 특수한 규제가 필요한 업종에서 다음과 같은 한계가 존재한다.

- **연령인증 페이지 커스터마이징 제한**: 아임웹이 제공하는 성인인증은 간편 인증(네이버, PASS, 토스 등), 휴대폰 인증, 카드 인증 3가지로 한정됨. 인증 UI/UX를 사업자가 자유롭게 수정할 수 없으며, 인증 플로우(예: 사이트 진입 시 전체 차단, 카테고리별 차등 인증 등)를 커스터마이징하기 어려움
- **상품별 미성년자 구매불가 설정은 가능**하나, 사이트 전체를 성인 전용으로 설정하거나 접속 자체를 차단하는 게이트 페이지 구현이 자유롭지 않음
- 템플릿 구조 자체를 수정할 수 없어 **전자담배 특화 레이아웃**(니코틴 함량 비교, 디바이스 스펙 필터 등) 구현이 난해함

## 2. 커스터마이징 한계 (HTML/CSS/JS 제한)

### 2.1 코드 수정의 구조적 제약

- **코드 위젯**을 통해 HTML/CSS/JS 삽입은 가능하나, 이는 프론트엔드 한정이며 **백엔드(서버 사이드) 커스터마이징은 불가**
- PHP, JSP, ASP 등 서버 사이드 언어 미지원
- 아임웹 업데이트 시 **사용자 커스텀 코드가 깨질 수 있으며**, 아임웹은 이에 대한 기술지원을 하지 않음
- 기본 위젯에 영향을 주는 코드는 업데이트 이후 정상 작동하지 않을 수 있음

### 2.2 기능 확장성의 한계

- **워드프레스의 6만+ 플러그인** 대비 아임웹 앱스토어는 극히 제한적인 외부 서비스만 연동 가능 (카카오싱크, 채널톡 등 일부)
- 특화 비즈니스 요구사항(재고 관리 고도화, 니코틴 성분 DB, 디바이스 호환성 검색 등) 반영 불가
- 전자담배 업종 특유의 기능(연령대별 접근 제어, 지역별 판매 제한, 규제 준수 자동화 등)을 자체 개발하여 탑재할 수 없음

### 2.3 외부 전문가 의존성

- 복잡한 커스터마이징은 크몽 등에서 아임웹 전문 개발자를 별도 고용해야 하며, 비용이 추가로 발생
- 아임웹 구조 변경 시 외주 작업이 무효화될 리스크 상존

## 3. 연령인증 지원 한계

### 3.1 제공되는 인증 방식

| 인증 방식 | 설명 |
|-----------|------|
| 간편 인증 | 네이버, PASS, 토스 등 민간 인증서 기반 |
| 휴대폰 인증 | 본인 명의 휴대폰 문자 인증 |
| 카드 인증 | 카드사 등록 정보 활용 |

### 3.2 전자담배 판매에서의 문제점

- **인증 우회 취약성**: 청소년의 전자담배 기기 온라인 구매 관련 게시글이 2020년 202건 → 2024년 1,338건으로 **5년 새 6배 이상 증가** (더퍼블릭 보도)
- 신분증 도용, 대리 인증을 통한 청소년 접근 사례가 지속적으로 발생
- 아임웹의 성인인증은 **상품 구매 단계에서만 작동**하는 구조로, 사이트 진입 단계에서의 차단(게이트 페이지)을 기본 제공하지 않음
- 합성니코틴 등 법적 회색지대 상품에 대한 별도 인증 로직 구현 불가

### 3.3 규제 준수 자동화 불가

- 전자담배 관련 법규 변경 시 인증 프로세스를 즉시 수정하기 어려움
- 지역별 판매 규제(특정 지자체 조례 등)에 대응하는 자동화 로직 구현 불가
- 판매 이력 기반 규제 준수 보고서 자동 생성 기능 없음

## 4. 결제/배송 연동 제한

### 4.1 결제 시스템

- PG 수수료: 결제 금액의 **2~3.5%** (모든 요금제 동일)
- PG 가입비 무료
- 해외 결제는 **Pro 요금제(월 24,000원) 이상**에서만 지원
- 자체 결제 게이트웨이 구축 불가

### 4.2 배송 연동

- **굿스플로** 기반 배송 추적 시스템 사용 (CJ대한통운, 우체국, 한진, 롯데, 로젠, 일양 등)
- 굿스플로 배송 추적 불가능 택배사의 경우 **송장번호 입력 후 10일 뒤 자동 배송완료 처리** — 실시간 추적 불가
- 네이버페이/카카오톡체크아웃에서 일부 택배사(드림택배, APEX 등) 선택 불가
- 서드파티 연동은 사방넷, 셀메이트, 플레이오토, 파스토 등 **공식 제휴사에 한정**
- 이지어드민, 이카운트 등은 비공식이라 **API 값을 직접 연동**해야 하며, 일반 운영자에게는 어려움

### 4.3 전자담배 특수 요구사항 미충족

- **로젠택배 API 직접 연동** 등 특정 택배사와의 커스텀 연동 불가
- 전자담배 배송 시 필요한 **성인 수령 확인** 프로세스 자동화 어려움
- 액상류 배송 시 필요한 위험물 라벨링/안내 자동화 기능 없음

- 회원 비밀번호 이전 불가로 인해 **전체 회원에게 비밀번호 재설정을 요청**해야 하는 운영 부담 발생

- 아임웹에서 구축한 디자인 자산은 타 플랫폼으로 이전 불가 → 매몰비용(sunk cost) 발생

## 6. SEO(검색엔진 최적화) 제한

### 6.1 구조적 SEO 한계

- 아임웹은 기본적으로 SEO 기본값(제목, 설명, URL 등)을 자동 적용하나, **고급 SEO 설정이 매우 제한적**
- **Yoast, 랭크매쓰** 같은 전문 SEO 플러그인 사용 불가
- 테크니컬 SEO, 오프 페이지 SEO에서 특히 약점

### 6.2 콘텐츠 SEO 취약성

- 아임웹은 **"콘텐츠 SEO에서 쥐약"**이라는 전문가 평가를 받음
- 블로그/콘텐츠 마케팅 기능이 워드프레스 대비 현저히 부족
- 구조화된 데이터(Schema.org) 커스터마이징 제한

### 6.3 성능(사이트 스피드) 문제

- **사이트 로딩 속도를 사용자가 컨트롤할 수 없음**
- 코드 감량, 캐시 최적화, 이미지 압축 등의 성능 최적화 작업이 제한적
- 사이트 스피드는 구글 SEO 핵심 랭킹 요소인데, 이를 직접 개선할 수 없음

### 6.4 전자담배몰 SEO 영향

- 전자담배는 **광고 규제로 인해 검색엔진 자연 유입이 매우 중요**한 업종
- SEO 최적화 한계는 곧 매출 감소로 직결
- 경쟁사 대비 검색 노출에서 열위에 놓일 수밖에 없음

## 7. 실제 운영자 불만 사례 및 이전 사유

### 7.1 비용 관련 불만

- Starter 요금제에서는 **상품 10개, 카테고리 3개**로 제한 → 전자담배몰 운영에 부족
- Pro 요금제(월 24,000원) 이상 필수이나, 기능 대비 비용이 높다는 평가
- 요금제 다운그레이드 불가로 인한 **비용 경직성**

### 7.2 기능 관련 불만

- "제대로 꾸며보려고 시도하면 이것도 안되고 저것도 안 된다"는 불만 다수
- 세밀한 커스텀 불가로 **고객 이탈율 증가, 구매전환율 하락** 우려
- 자동화 기능(재고 관리, 자동 발주, 마케팅 자동화 등)이 빈약

### 7.3 카페24 등 타 플랫폼으로 이전하는 주요 사유

| 이전 사유 | 상세 |
|-----------|------|
| 커스터마이징 한계 | 디자인/기능 자유도 부족 |
| 매출 성장 대응 불가 | 규모 확대 시 기능 한계 체감 |
| 데이터 활용 제한 | 고급 분석, CRM 연동 어려움 |
| SEO 한계 | 검색 노출 경쟁력 부족 |
| 비용 효율성 | 매출 대비 고정비 부담 |

### 7.4 전자담배몰 특수 불만

- 성인인증 프로세스를 업종 특성에 맞게 강화할 수 없음
- 전자담배 규제 변경 시 즉각 대응이 어려움
- 니코틴 함량, 디바이스 호환성 등 **업종 특화 필터/검색 기능** 구현 불가
- 전자담배 관련 **광고 규제 준수 자동화**(경고 문구 표시 등) 한계

### 결론

아임웹은 **소규모 브랜드의 빠른 런칭**에는 적합하나, 전자담배처럼 **강력한 규제 준수, 성인인증 고도화, 업종 특화 기능**이 필요한 쇼핑몰 운영에는 근본적인 한계가 있다. 특히 연령인증 커스터마이징 불가, 백엔드 개발 불가, SEO 제어 한계는 전자담배몰의 장기적 성장과 법적 리스크 관리에 심각한 장애 요인이 된다.
